# Security Policy

## Supported Versions

This project is currently maintained as a single evolving version.

Security fixes and updates are applied to the **main** branch.

| Version | Supported |
| ------- | --------- |
| main    | ✅ |

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please report it responsibly.

Please **do not create a public issue** for security vulnerabilities, as this may expose the problem before a fix is available.

Instead, report the vulnerability privately using one of the following methods:

- Email: jessicafram@example.com
- GitHub Security Advisory (if enabled)

When reporting a vulnerability, please include as much information as possible:

- Description of the issue
- Steps to reproduce the vulnerability
- Potential impact
- Suggested mitigation (if available)

## Response Process

After a vulnerability report is received:

1. The issue will be acknowledged within a few days.
2. The vulnerability will be analyzed and validated.
3. A fix will be implemented if the issue is confirmed.
4. Security updates will be applied to the **main branch**.

Responsible disclosure is appreciated to help maintain the security and reliability of this project.
